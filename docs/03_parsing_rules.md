# 03. 파싱 규칙 (LLM 금지 / 비용 0원)

## 1) 입력 예시
- `외주 50만원`
- `점심 1.2`
- `광고비 12만`
- `교재비 35000`
- `강의료 30`

---

## 2) 금액 추출 규칙

### 2-1. 지원 단위
- `원`, `천`, `천원`, `만`, `만원`
- 소수점 지원: `1.2만` → 12,000원

### 2-2. 기본 정책
- 단위가 없으면 원으로 간주 (`35000` → 35,000원)
- 단, **4자리 미만 숫자**(예: `30`, `500`)는 해석이 위험하므로:
  - `needs_confirm=true`
  - 금액 후보(`amount_candidates`)를 제안:
    - `[n 원]` 과 `[n * 1000 원]`
  - 예: `30` → 후보 `[30원] [30,000원]`

### 2-3. 정규식(권장)
> 구현 언어에 맞게 동일 동작을 보장하면 됨.

1) 소수점+만 단위:
- 패턴: `(?P<num>\d+(?:\.\d+)?)\s*(?P<unit>만|만원)`
- 금액: `int(float(num) * 10000)`

2) 정수+만원/만/천/천원/원:
- 패턴: `(?P<num>\d+)\s*(?P<unit>만원|만|천원|천|원)?`
- 단위별:
  - `만원`/`만` → `num * 10000`
  - `천원`/`천` → `num * 1000`
  - `원` 또는 None → `num`

### 2-4. 금액 추출 우선순위
- 가장 “의미 있는” 1개만 추출(MVP):
  1) 소수점+만
  2) `만원|만|천원|천|원` 붙은 수
  3) 단위 없는 가장 큰 정수 토큰(단, 4자리 미만이면 needs_confirm)

---

## 3) 수입/지출 분류 규칙

### 3-1. 키워드 사전
- 지출 키워드(우선):
  - `광고`, `임대`, `식비`, `점심`, `커피`, `교통`, `세금`, `보험`, `구독`, `장비`, `구매`, `결제`
- 수입 키워드:
  - `외주`, `강의`, `급여`, `정산`, `입금`, `수익`, `매출`

### 3-2. 판별 로직(결정적)
1) 지출 키워드가 포함되면 → `expense`
2) 아니면 수입 키워드가 포함되면 → `income`
3) 둘 다 없으면 → 기본 `expense`

### 3-3. 동점/충돌 처리
- 한 문장에 수입/지출 키워드가 모두 있으면:
  - 지출 우선(보수적) + `type_candidates`로 토글 제안

---

## 4) 파서 출력 규격(서버 내부 DTO)
```json
{
  "raw_text": "외주 50만원",
  "type": "income",
  "amount": 500000,
  "needs_confirm": false,
  "type_candidates": ["income","expense"],
  "amount_candidates": [490000, 500000, 510000],
  "reason": {
    "matched_amount": "50만원",
    "matched_type_keyword": "외주"
  }
}
```

## 5) 예외/에러 정책
- 금액을 찾지 못하면:
  - 저장하지 않고 `saved=false` 반환 + UI에 “금액을 찾지 못했어요” 노출
