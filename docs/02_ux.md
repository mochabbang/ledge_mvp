# 02. 화면/UX 정의 (구현 기준)

## 1) 온보딩 (`/onboarding`)

입력 항목:
- **이름 (닉네임)** — `user_metadata.display_name`에 저장, 거래 등록 시 `registered_by`로 사용
- **이번 달 순수익 목표 (원)** — `goals` 테이블에 upsert

CTA: "저장하고 시작하기" → `/dashboard` 이동
헤더 우상단 "목표 변경" 링크로 재진입 가능

---

## 2) 대시보드 (`/dashboard`)

### 2-1. 헤더
- 좌: `Ledge` (로고 텍스트)
- 우: `목표 변경` 링크 → `/onboarding`

### 2-2. 월 네비게이터
```
‹  2026년 3월  ›
```
- `‹` 클릭: 이전 월로 이동
- `›` 클릭: 다음 월로 이동 (현재 월 이상이면 비활성)
- 월이 바뀌면 데이터 재로드

### 2-3. KPI 카드 (5개)
상단 3개:
| 목표 | 순수익 | 달성률 |
|---|---|---|
| 3,000,000원 | 1,400,000원 | 46% |

하단 2개:
| + 총 수입 (초록) | − 총 지출 (빨강) |
|---|---|
| 2,000,000원 | 600,000원 |

### 2-4. 진행바
- `net / target` 비율로 표시
- 100% 초과 시 초과 표시

### 2-5. 일별 수입·지출 바 차트
- 내역이 없으면 숨김
- 가로 스크롤 (데이터 많을 때)
- 날짜별로 수입(초록)/지출(빨강) 쌍 막대
- **호버 툴팁**: 날짜 컬럼 전체에 `group hover` → "수입 50만원 / 지출 12천원" 형식

### 2-6. 입력창
```
날짜  [2026-03-04      ▼]
[예) 외주 50만원 / 점심 1.2만          Enter]
─────────────────────────────
[지출]  12,000원                      탭으로 전환
```

동작:
1. 날짜 선택기 (기본값: 오늘, 최대: 오늘 — 미래 선택 불가)
2. 텍스트 입력 즉시 클라이언트 파서가 실행 → 미리보기 노출
3. 미리보기 구성:
   - **수입/지출 토글 버튼** (클릭으로 전환, 색상 반영)
   - 감지된 금액
   - `needs_confirm=true`이면 "금액 확인" 뱃지
4. Enter 키로 제출 (typeOverride 포함하여 API 호출)
5. 저장 후 텍스트 초기화, 포커스 복귀

### 2-7. 내역 리스트 (날짜별 그룹)

**날짜 헤더:**
```
3월 4일 (화)                 +500,000  −12,000
```
- 오른쪽에 해당 일의 수입/지출 소계 표시

**내역 행:**
```
▲  외주 작업           +500,000원
   방금  [엄마]          순수익 ↑    ▾
```
- `▲` 수입(초록), `▼` 지출(빨강)
- 시간: `방금` / `N분 전` / `N시간 전` / `N일 전`
- `[닉네임]` 뱃지 (`registered_by`)
- `▾` 화살표: 인라인 수정 패널 열기/닫기

**인라인 수정 패널 (아코디언):**
```
내용  [외주 작업                    ]
      [수입]  [지출]
금액  [500000                       ]
      [−만원] [−천원] [+천원] [+만원]
[저장]                          [삭제]
```
- 삭제 버튼: 첫 클릭 → "삭제" / 두 번째 클릭 → "확인" (2단계 확인)
- 저장 클릭 시 데이터 재로드, 패널 닫힘

## 3) 공통 UX

### 토스트 알림
- 저장 성공: "저장됨"
- 삭제 성공: "삭제됨"
- 오류: "저장 실패" / "삭제 실패" / "금액을 찾지 못했어요"
- 3초 후 자동 소멸, 화면 하단 중앙 고정

### 로그인 (`/login`)
- Supabase Auth (이메일+패스워드)
- 미로그인 시 `/login` 리다이렉트

### 반응형
- 최대 너비 `max-w-lg`, 모바일 중심
