# RULES (절대 규칙)

## 1) 파싱은 LLM 금지
- 파싱은 **정규식 + 키워드 룰**로만 구현한다.
- 외부 유료 API 호출 금지(비용 0원 원칙).

## 2) 입력은 1줄, 저장은 즉시
- 사용자는 입력창에 1줄 텍스트만 입력한다.
- 입력 제출 시 서버는 즉시 저장한다.
- 저장 직후 **Undo(5초)**를 제공한다(클라에서 타이머 관리).
- Undo는 “삭제” 또는 “이전 상태 복구”로 동작한다(서버 API 제공).

## 3) 숫자 해석은 보수적으로
- 단위가 없고 **4자리 미만** 숫자(예: `30`)는 `needs_confirm = true`로 처리.
- `30`을 30,000원으로 자동 변환하지 않는다(오해 위험).
- 단, 입력 UX는 끊기지 않게: “30원이 맞나요 / 30,000원이 맞나요” 후보를 제안한다.

## 4) 기본 타입은 지출
- 키워드로 판별 불가 시 기본은 **expense(지출)**.
- 사용자는 제안 버튼으로 즉시 토글 가능해야 한다.

## 5) 데이터는 사용자별 완전 분리
- Supabase RLS 필수.
- 모든 테이블은 `user_id = auth.uid()` 조건을 강제.

## 6) 실시간 집계는 서버에서 계산
- 캐시 없이도 MVP는 충분(추후 최적화).
- `summary` 응답은 항상 서버에서 계산해 내려준다.

## 7) MVP 범위 고정(스코프 크립 방지)
- 카테고리 관리 화면, 자동 학습/추천, 영수증 OCR, CSV import 등은 MVP 제외.
